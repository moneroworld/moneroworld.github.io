how Bakerclassic release?

We believe that the the POW model has been the fairest way for initial BAK to be distributed. There is no pre-mining, no taxation, and no one gets something for nothing. You can download the compiled node program at https://github.com/Bakerclassiccoin/Bakerclassiccoin-Core/releases and obtain BAK through pow mining.

How to mine Bakerclassic(BAK)?

1 Install Ubuntu 18.04
2 sudo apt update
3 sudo apt upgrade
4 sudo apt install cmake
5 sudo apt install git
6 sudo apt-get install build-essential
7 sudo apt install software-properties-common
8 sudo add-apt-repository ppa:longsleep/golang-backports
9 sudo apt-get install golang-go
10 sudo apt update && sudo apt install build-essential cmake pkg-config libboost-all-dev libssl-dev libzmq3-dev libunbound-dev libsodium-dev libunwind8-dev liblzma-dev libreadline6-dev libldns-dev libexpat1-dev doxygen graphviz libpgm-dev qttools5-dev-tools libhidapi-dev libusb-dev libprotobuf-dev protobuf-compiler
11 Download gBAK from https://github.com/Bakerclassiccoin/Bakerclassiccoin-Core/releases
12 nohup ./gBAK --syncmode "full" --cache=1024 --maxpeers 1000 --datadir /home/ubuntu/data_BAK &
13 ./gBAK attach /home/ubuntu/data_BAK/gBAK.ipc
14 personal.newAccount()
15 miner.setBAKerbase(BAK.accounts[0])
16 miner.start(1)
the number in the () indicate the mining thread

Mapping Information

We will be releasing a new version of Baker. We will name the currently running Baker-Classic as Baker-Classic 1.0, and the new version will be named Baker-Classic 3.0.

Baker-Classic 3.0 will have the following changes:

Mining rewards will decrease from the current 10 BAK per block to 0.1 BAK per block.
Mining rewards will no longer be distributed to the project team.
After the release of Baker-Classic 3.0, coins on Baker-Classic 1.0 will be required to be mapped to Baker-Classic 3.0, with a mapping ratio of 10:1, meaning 10 BAK 1.0 will be exchanged for 1 BAK 3.0.

Baker-Classic 1.0 and Baker-Classic 3.0 systems will run concurrently for a period of time to ensure a smooth mapping process. After the mapping is completed, we will no longer maintain the Baker-Classic 1.0 system.

Specific exchange channels will need to be awaited for official Twitter announcements.
